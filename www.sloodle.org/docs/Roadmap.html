This roadmap is intended to give people an approximate idea of the kind of thing we're planning to work on.

This isn't at all fixed - we reserve the right to change our minds, and we're very open to suggestions. Notes about database changes these features will require are here .

Edmund Edgar, 2012-08-05

Some future version (Planned)

  1. Better support for multiple grids, eg. allow you to link multiple avatars per user, one on each grid.
  2. OpenSim Money module integration to allow the sloodle currency to be used as the currency of your grid.

Sloodle Version 2.2 (Planned)

  1. Integration of Freemail blogging tool.
  2. Set of generic game-like objects for rule-based simulations.
  3. Backpack / HUD.
  4. More, better quiz tools.
  5. Revisit permissions, and make sure we have an editable sloodle-specific setting where we're currently checking for things like "course update" permission.
  6. (?) Message queue system to get HTTP-in messages to people with NAT or firewalls in the way.
  7. NB Support for Moodle 1.9 will probably be dropped.

Sloodle Version 2.1 (In final beta testing)

  1. Support for Moodle 2 and existing Moodle 1.9.
  2. Full OpenSim support
  3. Shared media presenter.
  4. Shared media rezzer, with smart-phone UI and ability to save scenes.
  5. Awards points infrastructure.
  6. Scoreboard using ZZText.
  7. Infrastructure for gaming based on points
  8. NB Blog toolbar has been dropped due to issues with XYText in OpenSim and inconsistency with other tools.

Sloodle Version 1.2 (released June, 2011)

  1. Integrated tracker tools.

Sloodle Version 1.0

Tentatively scheduled for August

This was to be version 0.4.1 - but this will be in incremented to 1.0 instead to reflect stability of SLOODLE

  1. Presenter review
    1. Make editing more Moodle-like
    2. Add multi-file upload
  2. Quiz update
    1. Download one question at a time
    2. support for answer shuffling
    3. Support for question order shuffling
    4. Support multiple choice and true/false question types (only)
  3. Usability Evaluation - Review and improve Moodle pages
    1. Setting up the system and using it for students and teachers is simplified after systematic usability evaluations
    2. when you configure something there should be some clear concise instructions to return to SL and download the configuration. Instead there is a page of meaningless-to-the tutor data. We should probably still show that for the instructors that will want to know it, but demote it)
  4. Sets - Implement alternative arrangements
    1. Includes a redesign of the standard set
      1. Improve Set menu system (lots of people get lost with it, not a good start)
      2. Stop using hardcoded textures, replace with textures set from contents (e.g. Intercom)
      3. Layouts revision. Edmund has already done a lot of work here, he'll need help getting it tested and ensuring it works and usability testing (see his post here)
  5. Plugins and modules framework
    1. Ulster Tasks
    2. Due to dependencies that we have limited control over, Freemail will move to the Plug-Ins database
    3. iBank may also be posted in the plug-ins database
